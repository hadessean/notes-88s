Merge vs Rebase
===============
1. merge creates extra commit, rebase will not create extra commit
2. merge preserves history, rebase rewrites the history
3. rebase changes the commit id references

if shared branch go with merge, if private/local branch go for rebase
merge is the safest option...

git reset/revert
================
to undo the changes

soft -> it brings the changes from local to staging
mixed -> it brings the changes from local to workspace
hard -> delete the changes

reset is used only for local branches

revert is used to undo the changes in remote branches, it will not delete the commit, it will correct the code through extra commit...
